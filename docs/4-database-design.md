# Database Design

## Overview

The MVP database is designed to support:

* Authentication & Authorization
* Device Monitoring
* Metrics Collection
* Alert Management
* Incident Management
* Audit Logging


# Entities

The MVP consists of the following entities:

* Role
* User
* Device
* Metric
* Alert
* Incident
* AuditLog

# Relationships

| Relationship        | Description                                  |
| ------------------- | -------------------------------------------- |
| Role 1 → N User     | One role can be assigned to many users       |
| User 1 → N Incident | One user can be assigned multiple incidents  |
| User 1 → N AuditLog | One user can generate many audit log entries |
| Device 1 → N Metric | One device generates many metrics over time  |
| Device 1 → N Alert  | One device can generate many alerts          |

# Tables

## roles

### Purpose

Stores user roles and permissions.

### Columns

| Column     | Type      |
| ---------- | --------- |
| id         | UUID      |
| name       | VARCHAR   |
| created_at | TIMESTAMP |

### Example Entry

| id       | name  |      created_at      |
| -------- | ----- | -------------------- |
| role_001 | Admin |  2026-06-22 14:35:1  |

---

## users

### Purpose

Stores dashboard users.

### Columns

| Column        | Type      |
| ------------- | --------- |
| id            | UUID      |
| name          | VARCHAR   |
| email         | VARCHAR   |
| password_hash | VARCHAR   |
| role_id       | UUID      |
| created_at    | TIMESTAMP |

### Example Entry


| id       | name         | email               | role_id  | created_at          |
|----------|--------------|---------------------|----------|---------------------|
| user_001 | Tamanna Rath | tamanna@example.com | role_001 | 2026-06-22 14:35:12 |


---

## devices

### Purpose

Stores monitored infrastructure devices.

### Columns

| Column     | Type      |
| ---------- | --------- |
| id         | UUID      |
| hostname   | VARCHAR   |
| ip_address | VARCHAR   |
| status     | VARCHAR   |
| location   | VARCHAR   |
| created_at | TIMESTAMP |


### Example Entry

| id         | hostname      | ip_address | status | location  | created_at          |
|------------|-------------- |------------|--------|---------- |---------------------|
| device_001 | web-server-01 | 10.0.0.12  | Online | Mumbai DC | 2026-06-22 14:40:00 |


---

## metrics

### Purpose

Stores historical monitoring metrics.

### Columns

| Column       | Type      |
| ------------ | --------- |
| id           | UUID      |
| device_id    | UUID      |
| cpu_usage    | FLOAT     |
| memory_usage | FLOAT     |
| disk_usage   | FLOAT     |
| latency      | FLOAT     |
| packet_loss  | FLOAT     |
| recorded_at  | TIMESTAMP |

### Example Entry

| id         | device_id  | cpu_usage | memory_usage | disk_usage | latency | packet_loss | recorded_at         |
|------------|------------|-----------|--------------|------------|---------|-------------|---------------------|
| metric_001 | device_001 | 65.2      | 48.7         | 72.1       | 18.4    | 0.2         | 2026-06-22 14:45:00 |

---

## alerts

### Purpose

Stores alerts generated by monitoring rules.

### Columns

| Column     | Type      |
| ---------- | --------- |
| id         | UUID      |
| device_id  | UUID      |
| severity   | VARCHAR   |
| message    | TEXT      |
| status     | VARCHAR   |
| created_at | TIMESTAMP |


### Example Entry

| id        | device_id  | severity | message                  | status | created_at          |
|-----------|------------|----------|--------------------------|--------|---------------------|
| alert_001 | device_001 | Critical | CPU usage exceeded 90%   | Open   | 2026-06-22 14:46:13 |

---

## incidents

### Purpose

Stores operational incidents requiring investigation.

### Columns

| Column      | Type      |
| ----------- | --------- |
| id          | UUID      |
| title       | VARCHAR   |
| description | TEXT      |
| status      | VARCHAR   |
| assignee_id | UUID      |
| created_at  | TIMESTAMP |
| resolved_at | TIMESTAMP |

### Example Entry


| id           | title                  | description                  | status | assignee_id | created_at          | resolved_at |
|--------------|------------------------|------------------------------|--------|-------------|---------------------|-------------|
| incident_001 | Database Latency Spike | Investigating slow queries   | Open   | user_001    | 2026-06-22 14:47:55 | NULL        |

---

## audit_logs

### Purpose

Stores a history of important user actions.

### Columns

| Column      | Type      |
| ----------- | --------- |
| id          | UUID      |
| user_id     | UUID      |
| action      | VARCHAR   |
| entity_type | VARCHAR   |
| entity_id   | UUID      |
| created_at  | TIMESTAMP |

### Example Entry

| id        | user_id  | action            | entity_type | entity_id | created_at          |
|-----------|----------|------------------ |-------------|-----------|---------------------|
| audit_001 | user_001 | ACKNOWLEDGE_ALERT | Alert       | alert_001 | 2026-06-22 14:48:30 |

# Future Schema Evolution

The following entities may be introduced in later versions:

* Alert Rules
* Device Groups
* Incident Comments
* Notifications
* Maintenance Windows
* Teams
* Tenants
* SLA Policies

These are intentionally excluded from the MVP to keep the initial system simple and focused.
